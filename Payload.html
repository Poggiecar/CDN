BYPASS DE HOST

?t=at @ trick
host real = webhook.site. Si valida con contains/startsWith → bypass. EL más jugoso.
?t=atenc @ doble-encodeado
Testea si el nativo decodea dos veces antes de parsear el host.
?t=userinfopass userinfo con password
user:pass@host. Otra forma de meter MELI en userinfo.
?t=frag fragment confusion
MELI en el #fragment. Si el validador mira la string completa y no parsea → pasa.
?t=querymark marker en query del externo
MELI como param del propio webhook. Testea contains() bobo.
?t=backslash backslash trick
host real webhook.site. Parsers de Android/Java difieren con \.
?t=slashslash slash malformado
https:/ con una sola barra. Algunos parsers lo normalizan igual.
